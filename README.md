# PDF-to-PPTX Reconstruction Pipeline

A Python-based AI pipeline that converts flattened, non-editable PDF presentations (such as those generated by NotebookLM or standard image-based PDF exports) into fully editable Microsoft PowerPoint (`.pptx`) files. 

This tool leverages Document Layout Analysis (DLA) and Optical Character Recognition (OCR) to reconstruct slides, separating text, figures, and background colors into native PowerPoint objects.

## 🌟 Key Features

* **High-Fidelity Rasterization:** Converts PDF pages to high-resolution (300 DPI) lossless PNGs to ensure maximum Computer Vision and OCR accuracy.
* **Intelligent Layout Analysis:** Uses [Surya OCR](https://github.com/VikParuchuri/surya) to detect and classify complex page layouts (Titles, Text Blocks, Figures, Lists).
* **Text Extraction & Mapping:** Cropped text regions are processed via OCR and mapped directly into native, editable PowerPoint text boxes, matching the original bounding box positions.
* **Figure Extraction & Background Extraction:** Isolates chart and image assets and cropping them. The pipeline additionally computes the dominant background color of the original slide to recreate the original aesthetic.
* **Intermediate Representation (IR):** Decouples the computer vision tracking from the PPTX building process by saving a structured JSON configuration of all slide elements.

## ⚙️ How it Works

The pipeline operates in 3 core phases:
1. **Phase 1: Input Processing** - Normalizes variable PDFs and rasterizes them into a temporary staging directory using PyMuPDF.
2. **Phase 2: The Analysis Engine** - Deep Learning models (Surya) perform layout segmentation. A custom hybrid OpenCV contour splitter breaks monolithic figure/table regions into individual sub-panels.
3. **Phase 3: Reconstruction** - Translates pixel coordinates to English Metric Units (EMUs) and uses `python-pptx` to programmatically build the editable slide deck.

## 🚀 Installation

Ensure you have Python 3.9+ installed and a virtual environment set up.

```bash
# Clone the repository
git clone https://github.com/AsafTl/PDF2Slides.git
cd PDF2Slides

# Create and activate a virtual environment
python -m venv venv
source venv/bin/activate  # On Windows use: .\venv\Scripts\activate

# Install the required dependencies
pip install -r requirements.txt
```

*(Note: The first time you run the pipeline, it will automatically download the necessary HuggingFace deep learning weights for the Surya Layout and OCR models).*

## 📖 Usage

You can run the end-to-end pipeline using the provided CLI in `src/pipeline.py`. 

```bash
python -m src.pipeline "path/to/your/presentation.pdf"
```

### Optional Arguments:
* `--output` or `-o`: Specify a custom path for the generated `.pptx` file.
* `--staging`: Define a custom temporary directory for the high-res PNGs and cropped figure assets.
* `--keep-staging`: Prevents the deletion of the staging directory after the pipeline finishes (useful for debugging and reviewing OCR accuracy).

**Example:**
```bash
python -m src.pipeline "CRATERs_Blueprint_for_Immunotherapy_Success.pdf" --keep-staging
```
