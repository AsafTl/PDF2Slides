# PDF-to-PPTX Reconstruction Pipeline: Technical Blueprint

## Project Overview & Critical Limitations

The goal is to take a PDF file and convert it into a PowerPoint presentation file. The PDF file is in a flattened image format, where each page correspond to a slide. The PDF is generated by NotebookLM, It is all in an image format that include text as an image.

## Phase 1: Input Processing & Preparation

The objective is to standardize the input data. We must normalize variable compression and different color spaces into high-fidelity raster images.

* **PDF Ingestion:** Read the target PDF file and extract document-level metadata to determine the total page count and the native aspect ratio (typically 16:9 or 4:3).
* **Rasterization Engine:** Use a robust library like PyMuPDF (fitz) or pdf2image to render the PDF pages into bitmap arrays.
* **Resolution Setting:** Render the images at a minimum of 300 DPI to prevent severe degradation of the OCR engine accuracy.
* **Staging and Normalization:** Save the rendered images as lossless PNG files in a temporary directory to avoid JPEG compression artifacts.

## Phase 2: The Analysis Engine (The "Brain")

This phase uses computer vision and deep learning to deconstruct the flat image.



* **Layout Segmentation (Object Detection):** Pass the high-resolution PNG through a Document Layout Analysis (DLA) model like LayoutParser. The model must identify and classify regions into categories like Text, Title/Header, and Figure/Image, capturing the bounding box coordinates for each.
* **Background Extraction:** Calculate the dominant color of the pixels that fall outside all detected bounding boxes. If the variance is high, use an inpainting algorithm to fill in the bounding box areas, creating a clean background plate.
* **Optical Character Recognition (OCR):** Crop the original image using the bounding boxes classified as Text or Title/Header and pass these to an OCR engine like Tesseract OCR or PaddleOCR to extract the raw text string.
* **Formatting Estimation:** Attempt to extract metadata via the OCR engine regarding font weight and posture. Calculate the estimated font size by dividing the bounding box height by the number of text lines detected. Guessing the exact font family is highly unreliable and should be skipped.
* **Illustration Isolation:** Crop the original image using bounding boxes classified as Figure/Image and save these crops as individual PNG files in the temporary directory.

## Phase 3: Intermediate Representation (IR)

Create an intermediate JSON schema rather than writing directly to a PPTX file. This decouples the analysis engine from the presentation generator.

## Phase 4: PPTX Generation (Reconstruction) - Detailed Implementation

To translate the Intermediate Representation (IR) JSON into an actual `.pptx` file, you need to map pixel coordinates to PowerPoint's native measurement system and utilize a library like `python-pptx` (https://python-pptx.readthedocs.io/).

### 1. The Coordinate Translation (Pixels to EMUs)

PowerPoint does not use pixels for object positioning. It uses English Metric Units (EMUs). You must write a conversion function before placing any objects.

* 1 inch = 914,400 EMUs.
* Your pixel coordinates depend strictly on the DPI (Dots Per Inch) you used during the PDF rasterization in Phase 1.



To convert a pixel coordinate ($P$) to EMUs ($E$) given a specific $DPI$:
$$E = P \times \frac{914400}{DPI}$$

*Example:* If you rendered the PDF at 300 DPI, and a text box starts at an X-coordinate of 150 pixels: 
$150 \times (914400 / 300) = 457,200$ EMUs.

### 2. Initializing the Presentation

* **Load the Library:** Initialize `Presentation()` from `python-pptx`.
* **Set Slide Dimensions:** PowerPoint defaults to standard sizes (like 4:3 or 16:9), but you must explicitly set `prs.slide_width` and `prs.slide_height` using the EMU conversion of your IR's `base_width_px` and `base_height_px` to ensure the layout matches the original PDF exactly.

### 3. The Slide Construction Loop

Iterate through your JSON `slides` array. For each slide:

* **Add a Blank Layout:** `slide = prs.slides.add_slide(prs.slide_layouts[6])` (Layout 6 is typically completely blank in standard templates).
* **Set Background:** * *If a solid color:* Access `slide.background.fill.solid()` and set the `fill.fore_color.rgb` value.
  * *If a background plate image:* `python-pptx` has limitations with setting custom background images dynamically per slide without altering the slide master. The standard programmatic workaround is to use `slide.shapes.add_picture()` to insert the background plate image at `left=0, top=0` with `width=prs.slide_width` and `height=prs.slide_height`, effectively covering the slide background.

### 4. Placing Elements

Iterate through the `elements` array for the current slide. Convert all bounding box coordinates `[x, y, width, height]` to EMUs before passing them to the functions below.

* **Inserting Images (Figures):**
  * Use `slide.shapes.add_picture(image_path, left, top, width, height)`.
  * Pass the cropped PNG file path generated in Phase 2.
* **Inserting Text Boxes:**
  * Use `textbox = slide.shapes.add_textbox(left, top, width, height)`.
  * Access the text frame: `tf = textbox.text_frame`.
  * Insert the raw OCR text: `tf.text = element["text"]`.
  * **Applying Formatting:** You must iterate through the text paragraphs and runs to apply formatting. Set the font size using `run.font.size = Pt(estimated_size)`, apply bolding with `run.font.bold = True/False`, and enforce a standard fallback font (e.g., `run.font.name = 'Arial'`) to prevent cross-platform rendering issues.

### 5. Finalization and Cleanup

* **Save:** Commit the presentation object to a file using `prs.save('output_presentation.pptx')`.
* **System Cleanup:** Programmatically delete the staging directory (`/tmp/pdf_staging/`) containing the hundreds of temporary high-res PNGs and cropped figures to prevent disk bloat.
